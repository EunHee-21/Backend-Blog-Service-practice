기술 스택?
	보통 어떤 걸 구현하거나 내가 다룰 수 있는 기술들을 나열하는거죠.
	예를들어 나는 C,C++,C#,파이썬을 다룰수 있다 이렇게 하거나
	우리 회사에서 어떤걸 개발하는데 이러한 기술스택이 필요하다
	하면서 뭐 C++,자바,바,루비,파이썬이 요구된다 이런겁니다 (지식인 발췌~)

REST API ? (레스트 에이피아이)
	소프트웨어간 HTTP로 정보를 주고받는 부분이 있다~ (ex. 날씨앱이 기상청서버에 날씨정보를 요구함)
	그런데 이게 정보를 추가? 읽기만? 수정? 삭제 인지 한눈에 보기 어렵잖아?
	이걸 GET, POST등의 형식, 규칙을 정해서 쓰면 개발자들이 이해하기 쉬움 (RESTful~)
	한마디로 개발자들 사이의 암묵적인 약속, 형식 (마치 주문 폼처럼)
	서버에서 REST API로 요청을 보낼 때 HTTP란 규약에 맞춰 신호를 보냄.
	일반우편, 등기, 택배 등 다양한 방식이 있듯이, 이 HTTP로 요청을 보낼 때도 여러 메소드가 존재.
		읽기 = (GET) 
		추가 = (POST)
		전체 수정(ex. 학생의 이름 번호를 모두 변경) = (PUT)
		일부 수정(ex. 학생의 번호만 변경) = PATCH 
		삭제 = (DELETE)

Kotlin? (코틀린)
	젯브레인이라는 회사에서 만든 프로그래밍 언어인데, 케케묵은 자바랑 100퍼 호환가능!
	java bytecode(자바 바이트코드)로 변환이 되는데, 이게 다시 자바로 실행가능
	변경이 용이해서, 프론트 / 백엔드 / 안드로이드 등 다양한 분야에 써먹을 수가 있다!
		장점1. NULL SAFE : 변수는 null일수도 아닐수도 있습니다?
		장점2. coroutine(코루틴) : 많은 걸 한번에 실행하는 코드를 짤 수 있음.

mySQL? (에스큐엘)
	SQL -> 데이터베이스가 이해할 수 있는 언어
	ORM -> 파이썬 코드를 SQL코드로 바꿔주는 역할(번역기)
	영어로 되어있기 때문에 (SELECT, FROM…) 익히기 쉬움
	개념와 역할만 알고 있어도 도움이 됌.	번역기가 없이 SQL을 쓸수 있다면 위급상황 때 도움이 되겠지?

spring boot? (스프링부트)
	자바개발을 효율적으로 할 수 있는 프로그램
	프로그램을 만들기위한 여러 레시피가 존재해서 원하는 레시피만 찾고
	내것으로 만들 수 있음 (마치 햄버거 프렌차이즈 같이)

라이브러리와 프레임워크?
	다른 누군가가 쓴 코드로, 우리의 코딩 삶을 윤택하게 하기위해 가져다 쓰는 것들.
	라이브러리? 내가 필요로 할 때 도서관에서 자료 찾듯이 마음대로 빌려 쓰는 것. "내"가 라이브러리를 부름.
	프레임워크? (정해진) 프레임 내에서 작업을 해야 하는 것. "프레임워크"가 나를 부름.
	하지만 그 둘의 경계가 항상 뚜렷한 것은 아니다...

깃허브란?
	먼저, 깃 : 깃을 이용하면 누가 어떤 코드를 수정했는지 기록하고 추적할 수 있다. 
	많은 사람들이 함께 소프트웨어를 개발할 때 유용하다.
	깃허브 : 깃허브는 깃을 보다 편하게 이용할 수 있게 만든 호스팅 서비스
